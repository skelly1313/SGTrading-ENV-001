# DOCS/strategies/

Long-form, exploratory notes on candidate trading strategies.

This is the **scratchpad** — half-formed ideas, pros/cons, references.
Once a strategy is concrete enough to trade or backtest, promote it to a
formal spec under [`../../STRATEGIES/`](../../STRATEGIES/).

## Format suggestion

One file per strategy idea: `<short-name>.md`.

```markdown
# <strategy name>

## Thesis
Why this should work.

## Asset class / universe
What it trades.

## Signal
Entry / exit logic, even if vague.

## Risk
Stop, position sizing assumptions.

## Open questions
What we don't know yet.

## References
Papers, posts, prior backtests.
```
